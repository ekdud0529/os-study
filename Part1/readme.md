## 조민서


### 인터럽트가 무엇인가요?
> 인터럽트는 응용프로그램이나 하드웨어에서 예외가 발생해 처리가 필요한경우 cpu에게 이 예외를 알려 처리 하는것입니다. 

### CPU는 하드웨어중에 메모리에만 직접 접근이 가능하다. 만약 CPU가 다른 하드웨어(키보드, 마우스)에 접근하려면 어떻게 해야하나요?
> cpu에서 다른 하드웨어로 접근 하기위해서는 우선 키보드 또는 마우스 디바이스 컨트롤러에서 cpu로 인터럽트 신호를 보내야 한다.
인터럽트 신호가 오면 cpu는 메인메모리 최상단에 있는 인터럽트 벡터를 통해 주소를 찾아 인터럽트 서비스 루틴(함수)에 들어가 실행한다 (대기프로세스 → 실행프로세스 (이 과정을 디스패치라고함)). 그리고 프로세스 종료. 인터럽트 서비스 루틴에 들어가기전에는 스택에 복귀할 주소들을 저장해놔야함

### 멀티프로그래밍과 멀티태스킹의 차이점은 무엇인가요?
> 멀티프로그래밍은 cpu가 항상 프로그램 한개를 실행 할 수 있도록하여 cpu이용률을 높이는 것입니다. 운영체제는 여러 프로세스를 동시에 메모리에 올려두고 cpu는 이 중 하나를 처리합니다.
멀티프로그래밍을 하지 않을 경우 cpu는 유휴상태가 되어 cpu이용률이 낮아진다.
>
>멀티 태스킹은 멀티프로그래밍의 논리적확장입니다. 
멀티태스킹은 cpu가 메모리에 있는 여러 프로세스들을 빠르게 전환하여 빠른 응답시간을 제공해줍니다. 사람과 컴퓨터의 의사소통 즉 입출력은 대화식으로 하는데, 이때 사람의 (타자, 터치 등)은 느리지만 cpu는 매우 빠릅니다. 사람이 타자를 치는동안 남는 시간에 cpu는 다른 프로세스로 전환하여 실행합니다.
>
>멀티 프로그래밍과 멀티태스킹을 구현하기 위해서는 cpu스케줄링, 프로세스 스케줄링, 디스크관리, 가상메모리 등을 이용한다.

### 이중모드는 보호(보안)관점에서 어떤 기능을 제공하나요? 
> 이중모드를 통해 I/O장치보호, 메모리 보호, CPU보호를 합니다.
I/O장치보호: 예를 들어 이중모드가 없다면, 사용자 A와 사용자 B가 프린터를 이용합니다.
사용자 A가 프린터중에 사용자 B가 프린터 요청을 하면 프린터된 용지에는 
사용자 A 출력 용지 한줄
사용자 B 출력 용지 한줄
이런식으로 번갈아가며 출력이 될 것 입니다.

>메모리 보호: 사용자 A가 사용자 B 프린터에 저장된 내용을 출력 하려고 할 때 생기는 문제이다. 커널모드에서 메모리가 다르므로 인터럽트를 보낸다.

>CPU 보호: 사용자 프로그램에서 무한루프가 발생 했을 때 일정한 시간이 지났을경우 타이머에서 인터럽트를 보냄으로써 CPU독점을 방지한다.

### 운영체제의 목적은 무엇인가요? 
> 1. 사용자에게 인터페이스를 제공함으로써 하드웨어를 편리하게 사용한다.(사용 편의성, 사용자 관점)
> 2. 컴퓨터 시스템자원(CPU, 메모리, 파일시스템, 프로세스 등)을 효율적으로 관리한다. (자원할당자, 시스템 관점)
> 3. I/O장치와 사용자 프로그램의 부적절한 사용을 제어한다. (제어 프로그램, 시스템 관점)

### 캐시일관성 문제가 무엇인가요?
> 하드 디스크에 있는 F라는 파일 정수 A가 있다. 이 변수 A를 1증가 시키려고 한다.
우선 하드 디스크에 있는 파일(디스크 블록) F를 메인메모리로 가져오고, A를 캐시에 복사하고, A를 레지스터에 복사합니다. 내부레지스터에서 A+1증가 연산을 합니다. 이때 멀티태스킹인 경우 멀티프로세서를 이용하는데 각 cpu마다 각자의 로컬캐시를 가지고 있는데 이 캐시에 A라는 변수가 있으면 그 캐시에도 A는 A+1로 변해야한다. 하지만 변하지 않았는데 사용자가 변하지 않은 A값의 캐시를 참조하려고 할 때 생기는 동시성 문제이다.

### 캐시 용량의 가격이 하드디스크 용량의 가격과 같다면, 하드디스크를 버릴건가요? 버리지 않을 건가요? 이유도 같이 설명해주세요.
> 버리지 않는다. 캐시는 기본적으로 휘발성 메모리 이므로 전원이 꺼지면 메모리가 날라간다.
따라서 비휘발성 기억장치 하드디스크등을 이용해야 한다.

### 시스템콜을 이용해 운영체제에 접근할 때 운영체제에게 파일명 같은 매개변수를 전달해야 하는데 전달하는 방법을 말해주세요. 
>1. 매개변수를 레지스터에 직접 전달한다. But. 만약 레지스터보다 매개변수 개수가 더 많다면?
>2. 매개변수는 메모리 안의 블록이나 테이블에 저장하고, 테이블의 주소가 레지스터나 매개변수로 전달한다. (포인터)
>3. 매개변수는 프로그램에 의해 스택에 넣고, 운영체제에 의해 꺼내진다. (스택방법)

> 포인터방법과 스택방법은 매개변수의 길이나 제한을 받지 않는다.


### 프로세스 사이에서 통신하는 방법 2가지를 간단히 설명해주세요.

> 1. 공유 메모리
프로세스가 같은 메모리를 공유하는 경우, 공유 메모리 모델을 이용한다.
속도가 빠르다. 구현방식이 어렵다.
> 2. 메시지 전달
프로세스가 같은 메모리를 공유하지 않는 경우. 메시지 전달 모델을 이용한다.
컴퓨터간의 통신을 위해 메모리 공유보다 구현이 쉽다. 보호와 동기화 부분에서 여러 문제점이있다.

### 링커와 로더에 대해 설명해주세요.
> 링커는 오브젝트 파일을 하나의 이진 실행 파일로 결합해주는 역할을 하고, 로더는 이진 실행 파일을 메모리에 적재하여 CPU 코어에서 실행 가능한 상태로 만들어 줍니다.

## 조예은 
### 운영체제에 대해 설명해보세요
> 운영체제는 사용자와 응용 프로그램, 하드웨어가 서로 상호작용 할 수 있는 환경을 제공해주는 소프트웨어입니다. 또 운영체제는 여러 작업을 처리하는데 필요한 자원을 적절히 분배하여 할당해주는 자원 할당자이며, 컴퓨터의 부적절한 사용을 방지하기 위해 입출력 장치와 사용자 프로그램 수행을 제어하는 제어 프로그램입니다.

### 인터럽트 벡터란 무엇인가요
> 인터럽트를 처리하기 위해서는 적절한 인터럽트 서비스 루틴을 찾아 호출해야 합니다. 인터럽트 벡터는 인터럽트 서비스 루틴을 빠르게 찾아 호출하기 위해 일부 인터럽트 루틴의 주소를 저장하고 있는 테이블입니다. 

### 인터럽트 처리 과정에 대해 말씀해주세요
> 장치 컨트롤러에 이벤트가 발생하면 인터럽트 요청 라인에 신호를 보내 인터럽트를 발생시킵니다. 인터럽트가 포착되면 CPU가 수행 중이던 프로세스 정보를 저장 후 인터럽트 핸들러에게 제어권을 넘깁니다. 인터럽트 핸들러는 해당 인터럽트를 처리합니다. 인터럽트 처리 후  인터럽트 발생 전 상태로 복원하여 CPU가 제어권을 받으면 원래 수행하던 작업을 재개합니다. 

### 저장 장치의 종류와 특징에 대해 설명해보세요
> 저장 장치의 종류로는 레지스터, 캐시, 메인 메모리, 비휘발성 메모리, 하드 디스크 드라이브 등이 있습니다. 레지스터, 캐시, 메인 메모리는 1차 저장장치로 휘발성이고, 저장 용량은 적지만 액세스 속도가 빠르다는 장점이 있습니다. 다음으로 비휘발성 메모리와 하드 디스크 드라이브는 2차 저장장치로 대용량 데이터를 영구적으로 저장하는데 목적이 있습니다. 비휘발성 메모리는 전기적 저장장치로 기계적 저장장치인 하드디스크드라이브보다 비싸고 용량이 적지만 속도가 빠릅니다. 마지막으로 3차 저장장치인 광학 디스크, 자기 테이프 등이 있으며 저장 용량이 가장 크지만 가장 느립니다.

### 직접 메모리 액세스에 대해 설명해주세요
> 인터럽트는 비휘발성 저장장치 입출력과 같이 대량 데이터를 이동하는데에는 높은 오버헤드를 유발할 수 있습니다. 따라서 장치 컨트롤러가 CPU의 개입 없이 메모리와 해당 장치의 버퍼간에 직접적으로 데이터 블록을 주고받을 수 있도록 하는 것입니다. 바이트 단위로 인터럽트가 발생하는 것이 아니라 데이터 블록 단위로 인터럽트가 발생하므로 장치 컨트롤러가 전송 작업을 하는동안 CPU는 다른 작업을 수행할 수 있습니다.

### 클러스터형 시스템에는 비대칭형 클러스터링과 대칭형 클러스터링이 있습니다. 이 둘의 차이점에 대해 말씀해주세요
> 비대칭형 클러스터링은 하나의 호스트  서버가 활성 서버들을 감시하는 작업을 수행합니다. 활성 서버가 고장나면 호스트 서버가 활성 서버가 되어 고장난 서버의 작업을 수행합니다.
> 대칭형 클러스터링은 여러 서버가 응용 프로그램을 실행하고 서로 감시합니다. 사용 가능한 모든 하드웨어를 사용하기 때문에 비대칭형 클러스터링보다 효율적입니다.

### 모드 비트에 대해 설명해주세요
> 현재의 모드를 나타내기 위한 하나의 비트로, 커널 모드와 사용자 모드가 있습니다.모드 비트를 사용하여 운영체제를 위해 실행되는 작업과 사용자를 위해 실행되는 작업을 구분할 수 있습니다. 

### 사용자 모드에서 특권 명령을 수행하려고 하려면 어떻게 되나요
> 하드웨어는 이를 실행하지 않고, 불법적인 명령으로 간주하여 운영체제로 트랩을 겁니다.

### 시스템 콜이 무엇인가요
> 사용자 프로그램이 자신을 대신하여 운영체제가 수행하도록 지정되어 있는 작업을 하도록 운영체제에 보내는 요청입니다.

### 타이머란 무엇인가요
> 사용자 프로그램이 무한 루프에 빠지거나 시스템 서비스 호출에 실해하여 제어가 운영체제로 복귀하지 않는 경우가 없도록 지정된 시간 후 인터럽트가 발생하도록 하는 것입니다.

### 캐시 일관성 문제 대해 말씀해주세요
> 멀티 프로세서 환경에서는 CPU 별로 로컬 캐시를 가지고 있기 때문에 같은 데이터가 여러 캐시에 복사되어 있을 수 있습니다. 이런 경우 여러 개의 CPU가 모두 동시에 실행될 수 있으므로 한 캐시에 있는 데이터 값이 갱신되면 다른 캐시에 존재하는 해당 데이터 값이 모두 갱신되어야 합니다. 

### 링커와 로더의 역할에 대해 말씀해주세요
> 링커는 오브젝트 파일을 하나의 이진 실행 파일로 결합해주는 역할을 하고, 로더는 이진 실행 파일을 메모리에 적재하여 CPU 코어에서 실행 가능한 상태로 만들어 줍니다.

### 소스 프로그램 실행 과정
> 1. 소스 프로그램을 컴파일러로 컴파일하여 오브젝트 파일 생성
> 2. 오브젝트 파일을 링커를 사용하여 하나의 이진 실행 파일 생성
> 3. 로더가 실행파일을 메모리에 적재
> 4. 프로그램 실행

### 시스템 부팅 과정
> 1. 부트스트랩 프로그램 또는 부트 로더가 커널의 위치를 찾습니다.
> 2. 커널이 메모리에 적재되고 시작됩니다.
> 3. 커널은 하드웨어를 초기화합니다.
> 4. 루트 파일 시스템이 마운트 됩니다.

## 김다영
### 운영체제란?
> 컴퓨터 하드웨어를 관리하고 응용 프로그램 실행 환경을 제공하는 소프트웨어
  
### 운영체제의 역할은?
> 여러 응용프로그램 간의 하드웨어 사용을 제어 및 조정한다.
> 자원 할당자로 컴퓨터 시스템 동작 시 자원을 적절하게 사용할 방법 제공한다.
  
### 인터럽트란?
> 장치 컨트롤러가 CPU에게 신호를 보내 이벤트 발생을 알리는 것이다.
  
### 보조 저장 장치가 필요한 이유는?
> CPU가 명령을 실행하기 위해선 프로그램을 메인 메모리에 적재해야한다.
> 메인 메모리는 휘발성이고, 대용량 데이터를 영구히 저장하기엔 용량이 작다.
> 따라서 보조 저장장치에 프로그램을 저장한다.
  
### 프로세서란 무엇인가?
> 하나 이상의 CPU를 포함하는 물리적 칩
  
### 멀티 프로세서 시스템 종류 및 장단점
> 단일 코어 CPU가 있는 두 개 이상의 프로세서가 있다.
> 처리량이 증가하지만, 프로세서를 늘린 만큼 성량이 좋아지진 않는다.

### 부트 스트랩 프로그램이 무엇인가?
> 컴퓨터를 실행하기 위한 초기 프로그램으로 CPU 레지스터에서 장치 컨트롤러, 메모리 내용 등 시스템의 모든 측면을 초기화한다.
  
### 멀티 프로그래밍이란 무엇인가?
> 여러 프로그램을 메모리에 적재해두고, 하나의 프로세스가 대기 상태일 대 다른 프로세스 작업을 수행하는 시스템이다.
  
### 멀티 태스킹이란 무엇인가?
> CPU 스케줄링 알고리즘이 프로세스 간에 빠르게 전환해 작업하는 방식이다.
> 사용자 입장에선 마치 동시에 작동하는 것처럼 보인다.
  
### Dual-mode
> 사용자 프로그램이 함부로 시스템에 접근하지 못 하도록 사용한다.
> 잘못된 접근 시 Trap 발생한다.
  
### Trap은 무엇인가?
> 오류 또는 사용자 프로그램의 특정 요청으로 발생하는 인터럽트
  
### Timer란 무엇인가?
> 제어가 운영체제로 복귀하지 않는 것을 방지하기 위해 지정된 시간 후 인터럽트를 발생시키는 것
  
### 프로그램과 프로세스 차이
> 프로그램 : 디스크에 저장된 수동적 개체
> 프로세스 : 메모리에 적재된 능동적 개체
  
### 캐싱 과정
> 1. 특정 정보가 필요할 경우, 먼저 캐시에 그 정보가 있는지 조사
> 2. 정보가 있다면 그 정보를 캐시로부터 직접 사용
> 3. 없다면 메인 메모리 시스템으로부터 그 정보를 가져와 사용
> 4. 이 정보가 곧 다시 사용될 확률이 높다면, 캐시에 넣는다.
  
### 운영체제가 사용자에게 제공하는 서비스
> - 사용자 인터페이스
> - 프로그램 실행
> - 입출력 연산
  
### 운영체제가 시스템에게 제공하는 서비스 2가지
> - 자원 할당
> - 기록 작성
  
### System Calls란?
> 사용자 프로세스가 운영체제로부터 직접 제공받아야 하는 특정 서비스를 해달라고 요청하는 것
