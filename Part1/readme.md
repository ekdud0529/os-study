## 조민서

## 조예은 
### 운영체제에 대해 설명해보세요
> 운영체제는 사용자와 응용 프로그램, 하드웨어가 서로 상호작용 할 수 있는 환경을 제공해주는 소프트웨어입니다. 또 운영체제는 여러 작업을 처리하는데 필요한 자원을 적절히 분배하여 할당해주는 자원 할당자이며, 컴퓨터의 부적절한 사용을 방지하기 위해 입출력 장치와 사용자 프로그램 수행을 제어하는 제어 프로그램입니다.

### 인터럽트 벡터란 무엇인가요
> 인터럽트를 처리하기 위해서는 적절한 인터럽트 서비스 루틴을 찾아 호출해야 합니다. 인터럽트 벡터는 인터럽트 서비스 루틴을 빠르게 찾아 호출하기 위해 일부 인터럽트 루틴의 주소를 저장하고 있는 테이블입니다. 

### 인터럽트 처리 과정에 대해 말씀해주세요
> 장치 컨트롤러에 이벤트가 발생하면 인터럽트 요청 라인에 신호를 보내 인터럽트를 발생시킵니다. 인터럽트가 포착되면 CPU가 수행 중이던 프로세스 정보를 저장 후 인터럽트 핸들러에게 제어권을 넘깁니다. 인터럽트 핸들러는 해당 인터럽트를 처리합니다. 인터럽트 처리 후  인터럽트 발생 전 상태로 복원하여 CPU가 제어권을 받으면 원래 수행하던 작업을 재개합니다. 

### 저장 장치의 종류와 특징에 대해 설명해보세요
> 저장 장치의 종류로는 레지스터, 캐시, 메인 메모리, 비휘발성 메모리, 하드 디스크 드라이브 등이 있습니다. 레지스터, 캐시, 메인 메모리는 1차 저장장치로 휘발성이고, 저장 용량은 적지만 액세스 속도가 빠르다는 장점이 있습니다. 다음으로 비휘발성 메모리와 하드 디스크 드라이브는 2차 저장장치로 대용량 데이터를 영구적으로 저장하는데 목적이 있습니다. 비휘발성 메모리는 전기적 저장장치로 기계적 저장장치인 하드디스크드라이브보다 비싸고 용량이 적지만 속도가 빠릅니다. 마지막으로 3차 저장장치인 광학 디스크, 자기 테이프 등이 있으며 저장 용량이 가장 크지만 가장 느립니다.

### 직접 메모리 액세스에 대해 설명해주세요
> 인터럽트는 비휘발성 저장장치 입출력과 같이 대량 데이터를 이동하는데에는 높은 오버헤드를 유발할 수 있습니다. 따라서 장치 컨트롤러가 CPU의 개입 없이 메모리와 해당 장치의 버퍼간에 직접적으로 데이터 블록을 주고받을 수 있도록 하는 것입니다. 바이트 단위로 인터럽트가 발생하는 것이 아니라 데이터 블록 단위로 인터럽트가 발생하므로 장치 컨트롤러가 전송 작업을 하는동안 CPU는 다른 작업을 수행할 수 있습니다.

### 클러스터형 시스템에는 비대칭형 클러스터링과 대칭형 클러스터링이 있습니다. 이 둘의 차이점에 대해 말씀해주세요
> 비대칭형 클러스터링은 하나의 호스트  서버가 활성 서버들을 감시하는 작업을 수행합니다. 활성 서버가 고장나면 호스트 서버가 활성 서버가 되어 고장난 서버의 작업을 수행합니다.
> 대칭형 클러스터링은 여러 서버가 응용 프로그램을 실행하고 서로 감시합니다. 사용 가능한 모든 하드웨어를 사용하기 때문에 비대칭형 클러스터링보다 효율적입니다.

### 모드 비트에 대해 설명해주세요
> 현재의 모드를 나타내기 위한 하나의 비트로, 커널 모드와 사용자 모드가 있습니다. 모드 비트를 사용하여 운영체제를 위해 실행되는 작업과 사용자를 위해 실행되는 작업을 구분할 수 있습니다. 

### 사용자 모드에서 특권 명령을 수행하려고 하려면 어떻게 되나요
> 하드웨어는 이를 실행하지 않고, 불법적인 명령으로 간주하여 운영체제로 트랩을 겁니다.

### 시스템 콜이 무엇인가요
> 사용자 프로그램이 자신을 대신하여 운영체제가 수행하도록 지정되어 있는 작업을 하도록 운영체제에 보내는 요청입니다.

### 타이머란 무엇인가요
> 사용자 프로그램이 무한 루프에 빠지거나 시스템 서비스 호출에 실해하여 제어가 운영체제로 복귀하지 않는 경우가 없도록 지정된 시간 후 인터럽트가 발생하도록 하는 것입니다.

### 캐시 일관성 문제 대해 말씀해주세요
> 멀티 프로세서 환경에서는 CPU 별로 로컬 캐시를 가지고 있기 때문에 같은 데이터가 여러 캐시에 복사되어 있을 수 있습니다. 이런 경우 여러 개의 CPU가 모두 동시에 실행될 수 있으므로 한 캐시에 있는 데이터 값이 갱신되면 다른 캐시에 존재하는 해당 데이터 값이 모두 갱신되어야 합니다. 

### 링커와 로더의 역할에 대해 말씀해주세요
> 링커는 오브젝트 파일을 하나의 이진 실행 파일로 결합해주는 역할을 하고, 로더는 이진 실행 파일을 메모리에 적재하여 CPU 코어에서 실행 가능한 상태로 만들어 줍니다.

### 소스 프로그램 실행 과정
> 1. 소스 프로그램을 컴파일러로 컴파일하여 오브젝트 파일 생성
> 2. 오브젝트 파일을 링커를 사용하여 하나의 이진 실행 파일 생성
> 3. 로더가 실행파일을 메모리에 적재
> 4. 프로그램 실행

### 시스템 부팅 과정
> 1. 부트스트랩 프로그램 또는 부트 로더가 커널의 위치를 찾습니다.
> 2. 커널이 메모리에 적재되고 시작됩니다.
> 3. 커널은 하드웨어를 초기화합니다.
> 4. 루트 파일 시스템이 마운트 됩니다.
